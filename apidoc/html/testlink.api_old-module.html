reportTestCaseResult(testplan_id, status, testcase_id='', testcase_external_id='', build_id='', build_name='', notes='', guess=True, bug_id='', platform_id='', platform_name='', custom_fields={}, overwrite=False, **args)

source code 

Sets an execution result for a specified testcase.

Parameters:
  • testplan_id (int) - The internal ID of the testplan.
  • status (str) - The status of the testcase.
  • testcase_id (int) - (OPTIONAL) The internal ID of the testcase. If not given, external ID must be set.
  • testcase_external_id (int) - (OPTIONAL) The external ID of the testcase. If not given, internal ID must be set.
  • build_id (int) - (OPTIONAL) The internal ID of the build. If not given, the name name will be checked, otherwise the build with the highest ID is taken.
  • build_name (str) - (OPTIONAL) The name of the build. If not given, the buildid will be checked, otherwise the build with the highest ID is taken.
  • notes (str) - (OPTIONAL) Additional notes for the execution result.
  • guess (bool) - (OPTIONAL) If set, the API tries to 'guess' missing parameters (Default is True)
  • bug_id (int) - (OPTIONAL) The Bug ID referenced with the execution.
  • platform_id (int) - (OPTIONAL) The internal ID of the platform. If not given, platformname must be set, only(!) if platforms are available.
  • platform_name (str) - (OPTIONAL) The name of the platform. If not given, the platformid must be set, only(!) if platforms are available.
  • custom_fields (list of dicts (Example: [{'key': <FIELDNAME>, 'value': <VALUE>}])) - (OPTIONAL) Custom fields to set. Only custom fields, which are enabled during execution are allowed.
  • overwrite (bool) - (OPTIONAL) Enables overwriting of latest result. (Default is False)
Returns: dict
Success message
Decorators:
  • @proxy
  • @devkey

Note: The values of the custom fields must be given in the format, they are written to the database!

getTestCaseCustomFieldDesignValue(testcase_external_id, version, project_id, fieldname, *options, **args)

source code 

Returns informations and/or the value of a testcase's custom field at design level.

Parameters:
  • testcase_external_id (int) - The internal ID of the testcase.
  • version (int) - The version of the wanted testcase.
  • project_id (int) - The internal ID of the parent testproject.
  • fieldname (str) - The internal name of the custom field.
  • options - Additional options.
  • value - Returns only the value of the custom field. (Default configuration)
  • simple - Returns the value, the name of the custom field, the label and the type of the custom field.
  • full - Returns the definitions of all custom fields, includung their names, values.
